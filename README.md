## How I think

Seven patterns show up across everything I build:

1. **Risk-aware design first** - name the failure modes before the happy path
2. **Separation of concerns, obsessively** - research separates from build; directives separate from orchestration separate from execution
3. **Quality gates everywhere** - Last 20% Gate, Vendor matrix, Acceptance protocol, Notability gate
4. **IP is frameworks, not code** - AtriumCRM is 332K lines, but the lead is the frameworks behind those lines
5. **Faith as load-bearing infrastructure** - Delusional Belief is an operating thesis, not a sidebar
6. **Second-order thinking** - vendor validation checks the SLA, not just the API
7. **Adopts upstream, adds a layer** - absorb best-in-class patterns and re-engineer them for context

## Frameworks I've codified

- **Fulfillment Process** - Day-0 to handoff. Three anti-failures named and engineered against.
- **Prospect Flow** - An 8-step locked flow that separates research and planning from build.
- **Vendor Validation Playbook** - Five questions that catch third-party blockers in Discovery.
- **Three-Layer Agentic Architecture** - Directives + orchestration + execution. The pattern behind AtriumCRM, open-sourced as [directive-core](https://github.com/Asadulelah/directive-core).
- **Engagement Mapping** - per-prospect offer architectures: modular offers, tiered, one shape.
- **Carousel Content Engine** - HTML + PNG + caption production pipeline, open-sourced as [carousel-factory](https://github.com/Asadulelah/carousel-factory).
- **Compiled Truth + Vault Conventions** - The 9 conventions that keep a 1,985-page second brain maintainable.

More coming as I write them up.
