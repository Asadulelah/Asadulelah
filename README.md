## How I think

Seven patterns show up across everything I build:

1. **Risk-aware design first** — name the failure modes before the happy path
2. **Separation of concerns, obsessively** — research separates from build; directives separate from orchestration separate from execution
3. **Quality gates everywhere** — Last 20% Gate, Vendor matrix, Acceptance protocol, Notability gate
4. **IP is frameworks, not code** — AtriumCRM is 332K lines, but the lead is the frameworks behind those lines
5. **Faith as load-bearing infrastructure** — Delusional Belief is an operating thesis, not a sidebar
6. **Second-order thinking** — vendor validation checks the SLA, not just the API
7. **Adopts upstream, adds a layer** — absorb best-in-class patterns and re-engineer them for context
